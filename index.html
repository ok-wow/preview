okCOMPUTER Redesign

Design Screens & Decision Log — March 2026

Core Architecture Change: The HUD and panel are one continuous surface. The HUD pill transforms/morphs into the full panel view. Pill → card → full panel. One window that scales. Clicking the HUD expands it into the panel. The tray icon also triggers the same expansion. There is no separate "popover window" — the HUD IS the panel when expanded.

6. Key Design Decisions

DecisionChoiceRationale
Surfaces2 surfaces (HUD/Panel is one, Main Window)HUD morphs into panel — same window. Notification eliminated.
HUD expansionSingle morphing shapePill becomes card becomes panel. One continuous element.
Panel structureContextual sections, no tabs (except recording)Sections appear/disappear based on state.
Recording panelTwo tabs: Meeting + InsightsTranscript + scratchpad + proactive vault suggestions
Recording buttonPause (not mute)Mute is platform-specific. Pause stops all transcription.
Quick actionsFixed at bottom: Record / Note / ChatAlways accessible. Icons reinforce core functions.
OnboardingRPG agent creation + interview"Build your chief of staff." Name, avatar, voice, role, work style.
PermissionsHuman explanations during agent setup"So you can record meetings" not just "grant microphone"
Post-onboardingGuide to first real actionNot "you're set" — get them to build/explore/think to show power
First app openEarned via panel clickMain window never opens during onboarding
Progressive disclosureUsage-based, implicit tiersNo gates. Features introduced when relevant.
Insights destinationOpen in vaultSimpler. App has sharing functionality.
Panel minimizableYes, during recordingHUD pill shows recording status when minimized.
Screen shareHUD + panel hiddenPrivacy for meeting participants.
InterviewVoice or text, skippableAgent speaks first. Indi v7.0 principles. No empty prompt box.
