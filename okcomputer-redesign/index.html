The AI that thinks with you,
not just for you.

Redesigning okCOMPUTER — from 4 fragmented surfaces to a single morphing interface. HUD, panel, and notifications become one continuous shape.

Core Architecture: The HUD and panel are one window. The pill morphs into the full panel — pill → card → panel. Clicking the HUD or tray icon expands it. The notification window is eliminated.
Problem
4 fragmented surfaces. 183 audited issues. Users don't know where to look. Same info in multiple places. Onboarding teaches mechanics, not value.
Solution
2 surfaces: HUD/Panel (one morphing window) + Main Window. RPG "Build Your Chief of Staff" onboarding. Progressive feature disclosure.
Surfaces
HUD/Panel — The Dock. Always present. Morphs from pill to full panel.
Main Window — The Workspace. Deep content. Opens explicitly.

Design Decisions

DecisionChoiceRationale
Surfaces2 (HUD/Panel = one window + Main)HUD morphs into panel. Notification eliminated.
HUDSingle morphing shapePill → card → panel. One element.
PanelContextual sections, no tabsSections appear/disappear based on state.
RecordingTwo tabs: Meeting + InsightsTranscript + scratchpad + vault suggestions.
Rec buttonPause (not mute)Mute is platform-specific.
ActionsFixed bottom: Record / Note / ChatAlways accessible.
OnboardingRPG agent creation + interviewName, avatar, voice, role, work style.
PermissionsHuman explanations"So you can record meetings."
Post-onboardGuide to first actionNot "you're set" — get them building.
App openEarned via panel clickNever during onboarding.
DisclosureUsage-based, implicitNo gates.
InsightsOpen in vaultApp has sharing.
Screen shareHUD + panel hiddenPrivacy.
InterviewVoice or text, skippableAgent speaks first.
